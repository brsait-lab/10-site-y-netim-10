# PRODUCTION HARDENING & RELEASE CANDIDATE PHASE

## Amaç

Mevcut mimariyi değiştirme.

Yeni özellik geliştirme.

Yeni ekran geliştirme.

Bu çalışma yalnızca üretim öncesi son güvenlik, performans ve operasyonel dayanıklılık eksiklerini tamamlamak içindir.

Amaç:

Sistemi Release Candidate (RC) seviyesine çıkarmaktır.

---

# 1. RATE LIMITING

Tüm API endpoint'lerini incele.

Aşağıdaki endpoint'lere rate limit ekle:

* login
* register
* join-code validation
* notifications
* chat messages
* payment actions
* receipt upload
* vendor requests

Öneri:

Login:
5 istek / dakika

Mesaj:
30 istek / dakika

Bildirim:
20 istek / dakika

Aidat işlemleri:
10 istek / dakika

Brute force ve spam saldırıları engellenmeli.

---

# 2. FILE SECURITY HARDENING

Dekont sistemi için:

Kontrol et:

* MIME type doğrulaması
* Dosya uzantısı doğrulaması
* Maksimum boyut limiti
* Zararlı dosya yükleme riski

Kurallar:

İzin verilenler:

* jpg
* jpeg
* png
* pdf

Maksimum:

10 MB

Diğer tüm dosyalar reddedilmeli.

---

# 3. CLOUDFLARE R2 SECURITY

Dekontlar public olmamalı.

Kontrol et:

* bucket erişimi
* signed URL süresi
* erişim politikaları

Hedef:

Dosyalar yalnızca yetkili kullanıcılar tarafından görüntülenebilsin.

---

# 4. ENVIRONMENT SECURITY CHECK

Üretimde aşağıdaki env değişkenleri zorunlu olsun:

* JWT_SECRET
* SESSION_SECRET
* IBAN_SECRET
* DATABASE_URL
* R2_ACCESS_KEY
* R2_SECRET_KEY
* R2_BUCKET

Eksikse uygulama başlamasın.

Fail-fast mantığı uygulanmalı.

---

# 5. AUDIT LOG EXPANSION

Kontrol et:

* vendor request oluşturma
* vendor request tamamlama
* notification gönderme
* package teslim alma
* package teslim etme

Audit log'a düşüyor mu?

Eksikler tamamlanmalı.
